# Phase 999.1: UI Modernization + Year Field - Context

**Gathered:** 2026-03-21
**Status:** Ready for planning

<domain>
## Phase Boundary

Modernize the popup UI to match the high-fidelity mockup: new visual design system (Material Design 3 color tokens, Manrope + Inter fonts), redesigned header, gallery section, metadata form, and sticky download footer. Add a Year field to the metadata form and filename format. No library screen, settings screen, or bottom navigation bar in this phase.

</domain>

<decisions>
## Implementation Decisions

### Design system
- Replace current Tailwind color usage (blue-600 etc.) with Material Design 3 custom color tokens as defined in the mockup's `tailwind.config`
- Typography: Manrope for headings/labels (`font-manrope`), Inter for body/inputs — both bundled as WOFF2 files in `public/fonts/` (Google Fonts CDN is inaccessible in extension popups)
- Border radius: `rounded-lg` (0.5rem default), `rounded-xl` (0.75rem for cards/buttons)
- The full token set is defined in the mockup's `<script id="tailwind-config">` block — copy it verbatim into `wxt.config.ts` Tailwind config

### Header
- Sticky header with `backdrop-blur-md` and subtle shadow
- Left: `camera_enhance` Material Symbol icon (blue) + "Photo Extractor" in Manrope bold
- Right: "Scan Page" button — `bg-primary-container text-on-primary-container`, rounded-lg
- Header replaces the current stub header entirely

### Photo gallery section
- Content header: photo count badge (`bg-secondary-fixed`) + "Gallery" h2 in Manrope extrabold + "Select All" / "Clear All" links (right-aligned)
- 3-column grid (unchanged from current)
- Selected state: `ring-2 ring-primary` + blue overlay tint + filled `check_circle` icon (Material Symbols) in top-right corner using `bg-tertiary` (green)
- Unselected state: no ring, `opacity-80` with `group-hover:opacity-100`

### Metadata form
- Card treatment: `bg-surface-container-low p-5 rounded-xl` wrapping all fields
- Section label: "METADATA ASSIGNMENT" in uppercase tracking-widest, `text-on-surface-variant`
- Field order: Destination → Property/Vendor → [Category + Year side-by-side 2-col] → Notes
- All inputs: `bg-surface-container-lowest border-none rounded-lg p-3 text-sm focus:ring-2 focus:ring-primary/40`
- Category: `<select>` dropdown (locked options) with: Landscape, Accommodation, Dining, Activities — replaces the previous free-text datalist input
- Year: `<input type="number">` side-by-side with Category in a 2-col grid — defaults to current year (auto-detected via `new Date().getFullYear()` at runtime)
- Notes: `<textarea rows="2">` — replaces previous single-line input; placeholder: "Add a description for the advisor..."

### Year field behavior
- Included in filename: `destination_vendor_category_year_notes_index.ext` (when notes present) or `destination_vendor_category_year_index.ext` (when notes empty)
- `buildBasename` in `lib/naming.ts` must be updated to accept and include `year` between `category` and `notes`
- Year is persisted to `chrome.storage.local` and restored on popup mount, same pattern as destination/vendor/category
- Year is treated as optional for persistence fallback — if not stored, default to `new Date().getFullYear()`

### Sticky download footer
- Fixed bottom position (above nav if nav exists, but no nav in this phase)
- `bg-gradient-to-t from-surface via-surface to-transparent pt-8` gradient fade
- Button: full-width, `bg-primary-container text-on-primary-container font-manrope font-extrabold py-4 rounded-xl` with download icon + "Download N Photos"

### Bottom navigation bar
- **Omit entirely** — Library and Settings tabs are deferred to future phases
- No nav bar rendered in this phase

### Claude's Discretion
- Exact WOFF2 subset to download for Manrope and Inter (Latin subset is sufficient)
- Whether to load fonts via `@font-face` in a CSS file or inline in `main.tsx`
- Material Symbols icon loading strategy (subset or full CDN replacement — CDN won't work in extension; use static SVGs or a bundled icon approach)
- Exact transition/animation details on hover states
- How to handle the `<select>` styling cross-platform (Tailwind's `appearance-none` + custom chevron may be needed)

</decisions>
