# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Popup and Naming.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — plain Tailwind CSS 3.x (pre-existing) |
| Preset | not applicable |
| Component library | none — hand-rolled Tailwind components |
| Icon library | none — text labels and Unicode symbols only (no icon font to avoid popup bundle size overhead) |
| Font | system-ui (browser default — no custom font load) |

Source: `tailwind.config.js`, `entrypoints/popup/style.css` — no shadcn, no component library present in repo.
Chrome extension popup context: shadcn adds unnecessary JS weight; plain Tailwind is the established pattern.

---

## Popup Dimensions

Chrome extension popups have a fixed viewport. This is not a full web page.

| Constraint | Value |
|------------|-------|
| Popup width | 360px fixed |
| Popup max height | 600px (Chrome clips beyond this on most screens) |
| Scrollable region | Thumbnail grid only — form and actions are sticky/pinned |

Source: Phase 1 App.tsx established `w-72` (288px) as a starting point. Expanding to `w-[360px]` for Phase 3 to accommodate a 3-column grid and readable form fields. This is still within Chrome popup limits.

---

## Layout Structure

The popup is a single vertical stack with three regions:

1. **Header** — extension name, page title (read-only), image count badge
2. **Thumbnail Grid** — scrollable, 3 columns, fills available height
3. **Form + Actions** — sticky to bottom, naming fields + Download button

```
┌─────────────────────────────────┐
│ Header (48px fixed)             │
├─────────────────────────────────┤
│                                 │
│  Thumbnail Grid (scrollable)    │
│  [img][img][img]                │
│  [img][img][img]                │
│                                 │
├─────────────────────────────────┤
│ Form (destination/vendor/cat)   │
│ [Select All] [Download N]       │
└─────────────────────────────────┘
```

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between grid thumbnails, icon-to-label gap |
| sm | 8px | Input internal padding, label-to-input gap |
| md | 16px | Section padding, button internal padding |
| lg | 24px | Between form sections |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Header height |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Thumbnail touch/click target minimum: 44px height (accessibility baseline) — thumbnails are 108px wide x 80px tall so this is naturally satisfied
- Form fields: 36px height (input + padding) — between sm and md, use `py-2` (8px top+bottom) + `text-sm` line height

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 13px | 400 | 1.5 | Status messages, helper text, labels |
| Label | 12px | 500 | 1.4 | Form field labels above inputs |
| Heading | 15px | 600 | 1.2 | Popup header title "Photo Extractor" |
| Badge | 11px | 600 | 1.0 | Image count indicator, selection count |

Notes:
- 13px body (not 16px) is correct for Chrome extension popups — standard system UI density
- Tailwind classes: `text-xs` = 12px, `text-sm` = 14px — use `text-[13px]` for body and `text-[15px]` for heading
- No display-size text needed in a popup

## Color

All values use Tailwind utility classes. No custom hex tokens needed — Tailwind's built-in palette is the contract.

| Role | Tailwind Token | Hex Approx | Usage |
|------|---------------|------------|-------|
| Dominant (60%) | `white` / `gray-50` | #ffffff / #f9fafb | Popup background, form area background |
| Secondary (30%) | `gray-100` | #f3f4f6 | Thumbnail cell background, input backgrounds, header background |
| Accent (10%) | `blue-600` | #2563eb | Primary CTA button, selected thumbnail ring, checkbox fill |
| Destructive | `red-600` | #dc2626 | Error state text only — no destructive actions in this phase |

Accent reserved for exactly:
1. "Download N Photos" button background
2. Selected thumbnail border ring (2px `ring-2 ring-blue-600`)
3. Checkmark overlay icon on selected thumbnails
4. Focus ring on form inputs (`focus:ring-blue-500`)

Do NOT use blue on: unselected thumbnails, labels, header, form backgrounds, helper text.

## Interaction Contract

### Thumbnail Selection (UI-02, UI-03)

- Click an unselected thumbnail: adds blue ring + checkmark overlay, increments selection count
- Click a selected thumbnail: removes ring + overlay, decrements count
- "Select All" button: selects all thumbnails, label changes to "Clear All"
- "Clear All" button: deselects all thumbnails, label changes back to "Select All"
- Selection state is local to the popup session — not persisted

### Naming Form (NAME-01 through NAME-04)

- Three required fields: Destination, Vendor, Category
- One optional field: Notes (shown below required fields, labeled "Notes (optional)")
- All inputs are plain `<input type="text">`
- No dropdowns or autocomplete in Phase 3 (UX-01 persistence is v2 scope per REQUIREMENTS.md — but note: STATE.md lists "pre-fills last-used values" as a Phase 3 success criterion, see below)

**UX-01 clarification:** ROADMAP.md Phase 3 success criterion 5 states "Reopening the popup on a new page pre-fills the last-used destination, vendor, and category values." REQUIREMENTS.md lists UX-01 under v2. The executor should implement last-used pre-fill using `chrome.storage.local` — it is in the Phase 3 success criteria, treat it as in-scope.

- Field validation: required fields must be non-empty before Download is enabled
- No inline validation errors on blur — Download button simply stays disabled until all required fields have a value

### Download Action (UI-04, NAME-05)

- Download button label: "Download N Photos" where N is the current selection count
- When 0 selected or required fields empty: button is `disabled` + `opacity-50 cursor-not-allowed`
- On click: button shows "Downloading..." (disabled during operation)
- On success: button resets to "Download N Photos", status message shows "Saved N photos"
- On partial failure: status message shows "Saved X of N — Y failed"
- On complete failure: status message shows error copy (see Copywriting Contract)

### Loading State (popup open → content script response)

- On popup open: show skeleton grid (gray placeholder tiles, 9 tiles in 3x3 layout)
- Skeleton tiles fade in immediately — no blank white flash
- Real thumbnails replace skeletons as they load individually
- Timeout: if no response from content script within 5 seconds, show empty state

## States Reference

| Screen | Trigger | What Jennifer Sees |
|--------|---------|---------------------|
| Loading | Popup just opened | Skeleton grid (9 gray tiles), "Scanning page..." header |
| Populated | Content script returned images | Thumbnail grid, image count badge, form enabled |
| Empty | Page has no extractable images | Empty state illustration area + message |
| Selecting | Jennifer clicks thumbnails | Blue rings + checkmarks, count in button updates live |
| Form Filling | Jennifer types in fields | Inputs fill, Download button activates when all required fields have text |
| Downloading | Jennifer clicks Download | Button shows "Downloading...", thumbnails not interactable |
| Success | All downloads complete | Status message "Saved N photos", button resets |
| Error | One or more downloads failed | Status message with count + retry instruction |

## Accessibility Notes

- All interactive elements must have a visible focus ring (use `focus:outline-none focus:ring-2 focus:ring-blue-500`)
- Thumbnail selection state must be conveyed via `aria-pressed` or `aria-checked` on the button/checkbox element
- Download button disabled state uses `disabled` HTML attribute (not just visual opacity)
- Inputs must have associated `<label>` elements (not just placeholder text)
- Minimum tap target: 44x44px — all buttons and thumbnail cards meet this naturally at the specified sizes
