    In `entrypoints/popup/App.tsx`:

    1. Add import at top of file:

    2. Modify the `runDownloads` function. Currently (lines 303-310) the Promise.allSettled callback is:
       ```
       selectedUrls.map(async (url, i) => {
         const ext = deriveExt(url);
         await sendDownloadMessage(url, numberedBasenames[i], ext);
         dispatch({ type: 'DOWNLOAD_PROGRESS', done: ++done });
       }),
       ```

       Replace with:
       ```
       selectedUrls.map(async (url, i) => {
         // Derive extension from the ORIGINAL URL (before CDN rewrite) so the
         // file extension comes from the actual image filename, not CDN transform params
         const ext = deriveExt(url);
         const upscaledUrl = rewriteUrlForMaxResolution(url);

         if (upscaledUrl === url) {
           // No rewrite applied -- download directly
           await sendDownloadMessage(url, numberedBasenames[i], ext);
         } else {
           // CDN rewrite applied -- try upscaled URL first, fall back to original
           try {
             await sendDownloadMessage(upscaledUrl, numberedBasenames[i], ext);
           } catch {
             // Rewritten URL failed (404 or network error) -- silently try original.
             // If this also throws, Promise.allSettled catches it as 1 rejection.
             await sendDownloadMessage(url, numberedBasenames[i], ext);
           }
         }

         dispatch({ type: 'DOWNLOAD_PROGRESS', done: ++done });
       }),
       ```

    3. Add a comment above the import explaining: "CDN URL rewriting -- applies at download time only. Thumbnails in the grid use original extracted URLs. See lib/cdnRewrite.ts for supported CDN patterns."

    The key invariants:
    - deriveExt(url) is called with the original URL, NOT the rewritten URL
    - If upscaledUrl === url (no rewrite), skip the try/catch -- download directly
    - If upscaledUrl fails, fall back to original URL
    - If the fallback also fails, the error propagates to Promise.allSettled and counts as 1 rejection
    - The `dispatch({ type: 'DOWNLOAD_PROGRESS' })` call is AFTER the download (success or fallback success), so progress tracking is accurate
    - No double-counting: each URL produces exactly 1 settled result in Promise.allSettled

    Then update `tests/unit/cdnRewrite.test.ts` to add a "fallback behavior" describe block. These are unit tests of the rewriter, not integration tests of the full download flow:

    - Test: rewriteUrlForMaxResolution returns same reference for unknown URLs (use `toBe` not `toEqual` to confirm identity)
    - Test: rewriteUrlForMaxResolution returns a DIFFERENT string for known CDN URLs (confirming the caller can use `===` comparison to detect whether a rewrite occurred)

    Commit: `feat(04-02): integrate CDN URL rewriting into download pipeline with fallback`
