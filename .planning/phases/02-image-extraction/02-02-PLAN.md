    ```typescript
    import { extractImgTags } from '@/lib/extract/imgTags';
    import { extractCssBackgrounds } from '@/lib/extract/cssBackgrounds';
    import { parseSrcset } from '@/lib/extract/srcsetParser';
    import type { ImageResult } from '@/lib/extract/types';
    ```

    **Export `handleScanSession(port)` as a named export** so tests can call it directly without going through `defineContentScript`:

    ```typescript
    export function handleScanSession(port: chrome.runtime.Port): void {
      // ... scan logic here
    }
    ```

    Inside `handleScanSession`:

    1. Create a `Set<string>` called `seenUrls` for deduplication across initial scan and observer.

    2. Create a `blobCount` counter (number) initialized to 0. (Phase 3 will surface this count in the popup.)

    3. Set up `port.onMessage.addListener` that handles `{ type: 'SCAN_PAGE' }`:
       a. Call `extractImgTags()` and `extractCssBackgrounds()`
       b. Merge results, deduplicate by URL using seenUrls Set
       c. Count blob URLs encountered (the extractors already filter them out and log, but the count needs to be tracked for the SCAN_RESULT payload)
       d. Send `port.postMessage({ type: 'SCAN_RESULT', payload: { images: deduplicatedResults, blobCount } })`
       e. Start MutationObserver (see step 4)

    4. MutationObserver setup (inside the SCAN_PAGE handler, after initial results sent):
       - `new MutationObserver((mutations) => { ... })`
       - Config: `{ childList: true, subtree: true, attributes: true, attributeFilter: ['src', 'srcset'] }`
       - Observe `document.body`
       - For each mutation:
         - `childList`: check addedNodes for img elements (direct and descendants via querySelectorAll)
         - `attributes`: check if target is an img element with updated src/srcset
       - For each found img: apply same filtering as extractImgTags (100x100 min, skip SVG, skip blob, resolve srcset)
       - Deduplicate against seenUrls Set
       - Send each new unique image via `port.postMessage({ type: 'IMAGE_FOUND', payload: imageResult })`

    5. Helper function `processImg(img: HTMLImageElement, seenUrls: Set<string>): ImageResult | null` for the observer to reuse img filtering logic:
       - Check dimensions (naturalWidth/Height or width/height fallback)
       - Skip SVG (src ending .svg or type attribute)
       - Skip blob URLs (log to console)
       - Resolve srcset via parseSrcset
       - Check seenUrls dedup
       - Return ImageResult or null

    6. `port.onDisconnect.addListener(() => { observer.disconnect(); })` -- stops observer when popup closes.

    7. In the `defineContentScript` `main(ctx)` function body:
       ```typescript
       chrome.runtime.onConnect.addListener((port) => {
         if (port.name !== 'scan-session') return;
         handleScanSession(port);
       });
       ```

    **blobCount tracking:** The extractImgTags and extractCssBackgrounds functions log blobs to console but don't return a count. For Phase 2, track blobCount by scanning the DOM separately for blob src values before the main extract calls, OR accept that blobCount is 0 in the SCAN_RESULT for now and increment it when the observer encounters blob URLs. The simplest correct approach: after calling the extractors, do a quick `document.querySelectorAll('img[src^="blob:"]').length` count. This is an approximation (misses CSS blobs) but sufficient for Phase 2's console audit purpose.
