## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Camera icon too small to see clearly**
- **Found during:** Task 2 (human verification)
- **Issue:** Icon was w-6 h-6 (24px) and barely visible in the header
- **Fix:** Increased to w-8 h-8 (32px)
- **Files modified:** entrypoints/popup/App.tsx
- **Verification:** Build passes, icon visually larger in popup header
- **Committed in:** a653f4b

**2. [Rule 1 - Bug] Metadata form blocked by gallery thumbnails**
- **Found during:** Task 2 (human verification)
- **Issue:** Gallery grid using flex-1 expanded to fill all vertical space, pushing metadata form out of view
- **Fix:** Replaced flex-1 with explicit max-height: 210px + overflow-y-auto on the populated grid; removed flex-1 from all ThumbnailGrid states
- **Files modified:** entrypoints/popup/App.tsx
- **Verification:** Build passes, form visible below gallery without blocking
- **Committed in:** a653f4b

**3. [Rule 2 - Missing Critical] No error message on blank/new-tab scan**
- **Found during:** Task 2 (human verification)
- **Issue:** Clicking Scan Page on a new tab (chrome://newtab or about:blank) produced no feedback — extension silently did nothing
- **Fix:** Added URL scheme check in startScan; pages on chrome://, about:, chrome-extension://, edge:// or blank URLs dispatch SCAN_RESTRICTED, which renders "Can't scan this page" with an explanation
- **Files modified:** entrypoints/popup/App.tsx
- **Verification:** Build and all 149 tests pass
- **Committed in:** a653f4b
