## Component Inventory

New components required for this phase (all defined inline in App.tsx, mirroring existing pattern):

### LibraryPanel
Full-panel replacement for the main view. Same structure as SettingsPanel:
- Sticky header: back arrow (`arrow_back` SVG path, w-5 h-5) + "Library" title (text-base font-bold font-manrope)
- Header background: `bg-white/85 backdrop-blur-md shadow-sm shadow-slate-200/50` — identical to existing sticky headers
- Scrollable content area: `flex-1 min-h-0 overflow-y-auto`

### SortBar
Horizontal row of sort buttons below the sticky header. Controls which field records are sorted by.
- Buttons: "Recent", "Destination", "Vendor", "Category", "Year"
- Active button: `bg-primary-container text-on-primary-container` (matches SettingsPanel preset buttons)
- Inactive button: `bg-surface-container text-on-surface-variant hover:bg-surface-container-high`
- Button style: `py-1.5 px-3 rounded-lg text-xs font-medium font-manrope transition-colors`
- Container: `flex gap-2 px-4 py-3 overflow-x-auto` (horizontal scroll if buttons overflow 360px width)
- Default sort: "Recent" (newest savedAt first)

### LibraryRecord
Single row in the library list. Each record shows:
- Thumbnail: `w-14 h-14 rounded-lg object-cover flex-shrink-0` (56px square — compact but readable)
- Broken image fallback: `hidden w-14 h-14 rounded-lg bg-surface-container-high items-center justify-center text-on-surface-variant text-lg` showing "?" — mirrors ThumbnailCard pattern exactly
- Filename: `text-[13px] font-inter text-on-surface` — truncated with `truncate` class if overflowing
- Tag chips: destination, vendor, category, year rendered as inline chip elements
  - Chip style: `px-2 py-0.5 text-xs font-medium font-inter bg-secondary-fixed text-on-secondary-container rounded-full`
  - Renders all four fields; omits year chip if year field is empty
- Timestamp: `text-[11px] text-on-surface-variant` — relative date is acceptable ("Today", "Mar 20") but ISO substring is fine too
- Container: `flex items-start gap-3 px-4 py-3` with `border-b border-outline-variant` separator between records

### EmptyLibraryState
Shown when `records.length === 0` after loading:
- Container: `p-4 py-8 flex flex-col items-center justify-center text-center` — mirrors ThumbnailGrid empty state
- Heading: `text-[15px] font-semibold text-on-surface mb-1` — "No saved photos yet"
- Body: `text-[13px] text-on-surface-variant` — "Photos you download will appear here."

### LibraryIcon (navigation entry in PopupHeader)
Icon button added to PopupHeader alongside the gear icon.
- Uses Material Symbols `photo_library` SVG path (24x24 viewBox)
- Button style: `p-1.5 rounded-lg hover:bg-surface-container transition-colors`
- Icon size: `w-5 h-5 text-on-surface-variant`
- `aria-label="Library"`

├─────────────────────────────────────┤

│  [thumb] bali_four-seasons_pool_…   │
│          bali  four-seasons  ...    │  ← LibraryRecord rows

No sticky footer — the library is browse-only. No download button, no form.

## Interaction States

### Sort
- Clicking a SortBar button updates `sortBy` state immediately.
- Sort is client-side (Array.sort on the loaded records array).
- "Recent" sorts descending by `savedAt`. All other fields sort ascending alphabetically.
- No animation required — instant re-render is sufficient at this data scale.

### Navigation
- Library icon in PopupHeader sets `view = 'library'`. Back arrow in LibraryPanel header sets `view = 'main'`.
- Popup always opens on `'main'` view — library is not the default.
- When returning from library to main, all main view state (scan status, images, form fields) is preserved — `view` is local UI state only.

### Thumbnail loading
- Thumbnails use the `url` field from `SavedPhotoRecord` (original page URL at save time).
- `onError` handler hides broken `<img>` and shows "?" placeholder div — identical to ThumbnailCard pattern.
- No retry, no re-fetch. Stale URLs are a known limitation.

### Loading state
- `loadLibrary()` is async. While loading, show 3 skeleton rows:
  - Each skeleton: `flex items-start gap-3 px-4 py-3 border-b border-outline-variant`
  - Thumbnail placeholder: `w-14 h-14 rounded-lg bg-surface-container animate-pulse flex-shrink-0`
  - Text placeholder: `flex-1 space-y-2` with two `h-3 bg-surface-container-high rounded animate-pulse` bars
- 3 skeleton rows mirrors the 9-skeleton approach in ThumbnailGrid.

## Open Questions Resolved

The following open questions from RESEARCH.md are resolved for this phase based on codebase patterns and sensible defaults:

| Question | Resolution |
|----------|------------|
| Backfill for existing saves? | No backfill. Library shows records from `savedPhotos` log only. Empty state copy covers first-time use. |
| Duplicate handling? | Allow duplicates (log is a log, not a deduplicated index). Consistent with RESEARCH.md recommendation. |
| Display format? | Vertical list with thumbnail + filename + tag chips. Not a grid — metadata legibility is the priority. |
| Delete from library? | No delete in this phase. Out of scope per RESEARCH.md recommendation. |
