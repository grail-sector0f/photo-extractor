### Human Verification Required

#### 1. Library icon visible and navigates to Library panel

**Test:** Load the unpacked extension in Chrome. Open the popup on any page. Look for a photo-grid icon in the header to the left of the gear icon.
**Expected:** Icon is visible; clicking it replaces the main view with the LibraryPanel (sticky header with back arrow and "Library" title, FilterBar with 3 dropdowns, content area).
**Why human:** Visual rendering and click behavior in a Chrome extension popup cannot be verified without a browser.

#### 2. Downloaded photos appear in Library after download

**Test:** Scan a travel site page, select images, fill in metadata (destination, category, year at minimum), click Download. After completion, open the Library.
**Expected:** Each downloaded image appears as a row with thumbnail, filename, and tag chips showing the metadata values entered.
**Why human:** Requires real download flow through chrome.downloads API and chrome.storage.local persistence; cannot mock end-to-end in unit tests.

#### 3. FilterBar dropdowns filter the record list

**Test:** With multiple records in the Library showing different destinations, select a specific destination from the Destination dropdown.
**Expected:** Only records matching that destination remain visible. Selecting the default ("Destination") restores all records.
**Why human:** React state interaction and dropdown change events require browser rendering.

#### 4. Broken thumbnail shows '?' placeholder

**Test:** If any library records have URLs that are now unreachable (e.g., a hotel page you're not currently browsing), open the Library and look at those records.
**Expected:** The image area shows a "?" character on a gray background instead of a broken image icon.
**Why human:** Requires the browser to attempt image load and fire the `onError` event handler.

#### 5. Per-record trash button removes a record

**Test:** In the Library, click the trash icon on any record.
**Expected:** The record disappears immediately from the list. Re-opening the Library (navigate away and back) confirms it is gone.
**Why human:** Requires confirming that `removeFromLibrary` persisted the deletion to chrome.storage.local, not just removed it from local React state.
