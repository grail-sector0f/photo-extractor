# Phase 999.3 Plan 2: Library UI Summary

One-liner: React LibraryPanel with SortBar (5 sort fields), record rows with thumbnails and tag chips, skeleton loading, empty state, and 9 unit tests for the sort comparator.

## What Was Built

Task 1 (complete): Built the full Library UI in `entrypoints/popup/App.tsx` and sort tests in `tests/unit/library-sort.test.ts`.

- **compareRecords**: Module-level exported comparator. `savedAt` sorts descending (newest first via `b.savedAt.localeCompare(a.savedAt)`). All other fields sort ascending alphabetically.
- **SortBar**: Horizontal scroll row of 5 buttons (Recent / Destination / Vendor / Category / Year). Active button uses `bg-primary-container text-on-primary-container`. Inactive uses `bg-surface-container`.
- **LibraryRecord**: Row with 56px thumbnail, `onError` "?" placeholder, truncated filename, tag chips for destination/vendor/category/year, and `YYYY-MM-DD` date.
- **EmptyLibraryState**: "No saved photos yet" + "Photos you download will appear here."
- **LibrarySkeletons**: 3 animated skeleton rows shown while `loadLibrary()` is in flight.
- **LibraryPanel**: Full-panel view with sticky header (back arrow + "Library"), SortBar, and scrollable record list. Loads records on mount via `loadLibrary()`, sorts client-side.
- **PopupHeader update**: Added library icon button (photo_library SVG path) with `aria-label="Library"` before the gear icon. Added `onLibrary` to `PopupHeaderProps`.
- **App update**: View state union extended to `'main' | 'settings' | 'library'`. Library view block added before settings check. `onLibrary={() => setView('library')}` passed to PopupHeader.
- **Sort tests**: 9 tests covering savedAt (newest first), destination/vendor/category/year (alphabetical ascending), and two full-array integration tests.

Task 2 (checkpoint): Human verification in Chrome — awaiting user sign-off.
