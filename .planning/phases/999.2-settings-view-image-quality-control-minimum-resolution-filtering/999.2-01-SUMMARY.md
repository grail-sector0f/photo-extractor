- Created `lib/settings.ts` as the single source of truth for all user-configurable settings — AppSettings interface, DEFAULT_SETTINGS, MIN_DIMENSION_PRESETS, loadSettings (merge-over-defaults), and saveSettings

- Wired settings throughout the popup: loaded on mount (single storage read), minDimension passed in SCAN_PAGE message, skipGifs applied to SCAN_RESULT/IMAGE_FOUND before dispatch, cdnUpscalingEnabled gates CDN rewrite at download time
- 16 new unit tests for settings module (TDD), 3 new content script tests; full suite 168 tests green

## Decisions Made

- DEFAULT_SETTINGS values are per user decisions: minDimension=150 (Medium), skipGifs=false, cdnUpscalingEnabled=true
- GIF filter applied in port.onMessage callback (outside reducer) — keeps reducer pure; deriveExt is a pure function imported at module scope
- loadSettings uses `{ ...DEFAULT_SETTINGS, ...(stored.settings ?? {}) }` spread pattern — safe for future field additions without migration
- Settings loaded in the existing mount useEffect alongside prefill, not a separate effect — avoids two separate chrome.storage.local.get calls (per RESEARCH.md Pitfall 2)
- cdnUpscalingEnabled toggle leverages the existing `upscaledUrl === url` identity check as the fast path — no new branching overhead needed
